Privacy Policy

ShortcutStudio is designed to be privacy-minded. This page explains what data is stored, where it is stored, and how to revoke access.

What ShortcutStudio stores

ShortcutStudio stores the minimum data required to connect your third‑party accounts and execute requests from the app and Shortcuts:

  • OAuth tokens (access tokens and, when issued by the provider, refresh tokens) for connected services.
  • Connector configuration (which services you connected, display names, and connection status).
  • Diagnostics and connector issue summaries (optional, user-controlled; intended to help you understand failures like “auth expired” or “offline” without exposing secrets).

ShortcutStudio does not intentionally collect or store your passwords for these services.

Where data is stored

  • Tokens are stored in the system Keychain (iOS/iPadOS/macOS). The operating system provides encryption-at-rest and access controls for Keychain items.
  • Non-sensitive app state (for example, connection display names and issue summaries) is stored using platform-appropriate app storage. Some connector issue summaries are stored in an App Group container so the app and Shortcuts extension can share the same status.

Authentication method

  • ShortcutStudio uses ASWebAuthenticationSession for OAuth sign-in.
  • ShortcutStudio does not use embedded webviews for sign-in.

Requested scopes and why

ShortcutStudio requests only the scopes required to support the connector features that the app exposes.

GitHub

  • Scope: repo
  • Why: read repository issues for “List Issues”.

Jira (Atlassian)

  • Scope: read:jira-work
  • Why: search issues using JQL for “Search Issues”.

Notion

  • Scope: read_content
  • Why: search pages/databases for “Search”.

Todoist

  • Scope: data:read
  • Why: read tasks for “List Tasks”.

Spotify

  • Scope: user-read-recently-played
  • Why: fetch recently played tracks for “Recently Played”.

Disconnecting and revoking access

ShortcutStudio supports disconnecting a service at any time:

  • Disconnect in-app: removing a connection deletes its tokens from Keychain immediately and updates the UI state.
  • Revocation (when supported by the provider): the app will attempt to revoke the token server-side during disconnect.
  • Provider-side revocation: you can always revoke ShortcutStudio’s access from the provider’s security/application settings page; after revocation, ShortcutStudio will surface an “auth expired/needs attention” state until you reconnect.

Diagnostics and privacy

If diagnostics are enabled, ShortcutStudio records high-level request outcomes (success/failure, short cause, timestamps) and exports anonymized traces. Secrets (tokens, authorization codes) should not be included in exports.
